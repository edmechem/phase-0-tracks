Here's another aside: these are my "Release 4: Reflect" comments.

  • What is the purpose of HTML5 semantic tags? Which ones do you see as most useful?
    The HTML5 semantic tags exist to provide an indication of the meaning, type & purpose of various sections of the page. They all seem useful, but if I had an RSS Feed Reader subscription to some site or blog, the <article> tag would be particularly useful, in that it would tell the feed reader what part of the page it should pay attention to, and what part (ouside the article element) can be ignored.
  • What is the order the browser uses to display elements on a page?
    We haven't learned CSS yet, so without and style indicating a different placement, the browser basically displays elements in order, top to bottom.
  • What did you learn about Sublime in this challenge? Do you think you'll create more handy snippets at a later date?
    Sublime's snippets are awesome. My guide helped me walk thru the process of learning the syntax of Sublime snippets. I definitely think it's a useful feature that I'll find various other uses for as we progress.
  • What concept in this challenge was most difficult for you? How can your understanding of this concept?
    Understanding the proper nesting of the HTML5 semantic tags was the difficult concept for me to understand. The HTML5 Element Flowchart on HTML5 Doctor was pretty great; although it neglected to include a <header> tag for the entire page (which should come before nav), and a <main> element after nav, encompassing the article element(s).
